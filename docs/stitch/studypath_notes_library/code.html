async/await — tratamento de erro

Pontos principais

  • Sempre utilize try...catch em funções assíncronas para capturar erros de rede ou lógica.
  • Considere o uso de Promise.allSettled quando houver múltiplas requisições independentes.
  • Erros não capturados podem derrubar o processo Node.js se não tratados globalmente.

Cuidado:

Evite "vazar" detalhes técnicos do erro para o cliente final. O catch deve logar o erro original mas retornar uma mensagem amigável.

Exemplo prático

async function getUserData(id) {

try {

const response = await fetch(`/api/users/${id}`);

if (!response.ok) throw new Error('User not found');

// Sucesso

return await response.json();

} catch (err) {

console.error('Logging service error:', err);

throw err;

}

}
